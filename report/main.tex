\section{Задание}

\begin{tcolorbox}
    Автоматически построить рефераты текстовых документов. \\
    Ввод: Массив текстов.
    Вывод: Массив рефератов.

    Максимальный размер каждого из рефератов — 300 символов (включая пробельные). Если размер реферата превышает указанный порог, то будут оцениваться только первые 300 символов. Тривиальное решение не допускаются (первые 300 символов документа).

\end{tcolorbox}

\section{Описание пайплайна}
Для построения рефератов будет использоваться экстрактивный подход с использованием tf-idf. 
Это значит, что реферат будет состоять из предложений исходного текста. 
Данный подход позволяет избежать проблем генеративных подходов на нейросетях, которые склонны к придумыванию, перифразу и другим галлюцинациям, из-за которых смысл реферата может оказываться совершенно другим.
Уточним терминологию: Документ - предложение в тексте, Корпус - весь текст, на который мы генерируем реферат. 

Пайплайн:
    \begin{itemize}
        \item Из всех предложений убрать стоп слова, частицы, предлоги, междометия, союзы.
        \item Привести все слова из всех предложений начальной форме. (\emph{использован PyMorphy3})
        \item Подсчитать частоту встречания каждого слова во всем корпусе.
        \item Подсчитать частоту встречания каждого слова в текущем предложении(tf).
        \item Вычислить $idf = log(\frac{\text{всего документов в корпусе}}{\text{частота встречания слова t из документа d во всем корпусе}})$
        \item Вычислить $tfidf = tf * idf$ для каждого слова, затем каждого предложения сложить $tfidf$ всех слов в нем. Это и будет весом предложения.
        \item * Эвристика: обычно в новостях, статьях первые предложения довольно информативные, умножим их вес на 1.5 для первого предложения и на 1.2 для следующих двух.
        \item Отсортируем все предложения из корпуса по их весу, возьмем столько предложений, чтобы не вылезти за границу в 300 символов.
        \item Отсортируем в изначальном порядке, чтобы не было потери смысловой последовательности.
    \end{itemize}
